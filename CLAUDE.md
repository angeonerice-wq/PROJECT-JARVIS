# PROJECT-JARVIS 運用ルール

## PRマージの判断基準
- 軽微な変更(タイポ修正、コメント、細かいUI微調整、`flutter analyze`のみで検証可能なもの、改行コードのみの差分)
  → 確認なしでマージを進めてOK
- 重要な変更(認証・Firestore・セキュリティルール・新機能・データ構造/ロール権限に影響するもの)
  → 動作確認を促してからマージ判断する

## コマンド承認の目安
- 読み取り専用コマンド(`git status`, `git branch`, `git log`, `gh pr view`, `gh auth status` など)
  → 常に許可扱いでOK
- 書き込み・削除・push・force系コマンド → 都度確認

## 絶対に自動化しない操作(常に人間の確認が必要)
- 課金・支払いが発生する操作(有料プラン変更、課金APIの大量呼び出しなど)
- 本番データの完全削除(Firestoreのhard delete、trashの完全削除など)
- セキュリティルールの緩和・権限拡大
- 外部への公開・共有設定の変更

これらは他のいかなるルールに関わらず、必ずMakotoに確認を取ってから実行すること。
